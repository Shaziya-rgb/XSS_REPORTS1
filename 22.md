## Title
Stored Cross-Site Scripting (XSS) in "Site title, Site Description, Welcome message, Footer text" fields

## Vulnerability Type
Reflected XSS

## Summary
The `Site title, Site Description, Welcome message, Footer text` parameter on the page reflects user input directly inside a JavaScript analytics tracking script without any escaping or sanitization. This means if you break out of the string context with a quote and inject JavaScript, it executes immediately. No storage involved, the payload fires the moment the page loads with the crafted URL.

## Vulnerable Endpoint
```
https://kzlabs.com/22.php?Site title, Site Description, Welcome message, Footer text=
```
## Steps to Reproduce
1. Go to `https://kzlabs.com/22.php` 
2. Submit this as the search term or visit the following URL directly:
```
https://kzlabs.com/22.php?
```
3. The page loads and a JavaScript alert box pops up displaying `1` confirming the payload broke out of the JS string and executed.

---
## Payload Used
```
"><sCrIpt>alert(1)</sCrIpt>
```

## Impact
- An attacker can craft a malicious URL and send it to and the moment they click it the script executes in their browser
- Steal session cookies of anyone who clicks the link giving the attacker full access to their account
- Can be used to redirect victims to fake login pages that look identical to Equifax to harvest credentials
- The vulnerability exists inside an analytics script which shows the dev team never considered it as an injection point making it easy to miss during code review

## Remediation
1. Filter out HTML tags like `<script>`, `<img>`, `<svg>` from the search field before reflecting anything back to the page
2. Filter out JavaScript methods like `alert()`, `confirm()`, `prompt()` so even if a quote slips through the method won't execute
3. Use Cloudflare as they have so many WAF rules that almost all XSS payloads will be blocked automatically before even reaching the server
